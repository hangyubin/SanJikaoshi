# 基层医院三基考试系统使用指南

## 1. 系统概述

基层医院三基考试系统是基于智能考试系统开发的专业医疗人员三基（基础理论、基本知识、基本技能）考试平台。该系统支持用户注册登录、考试管理、题目管理、在线考试、成绩统计、科室管理、证书管理和学习资源管理等功能，采用现代化的技术栈，具有良好的可扩展性和可维护性。

## 4. 系统使用指南

### 4.1 管理员操作指南

#### 4.1.1 科室管理
1. 登录系统后，点击左侧菜单的「管理员功能」→「科室管理」
2. 在科室管理页面，可以查看所有科室列表
3. 点击「新增科室」按钮，填写科室信息，包括科室名称、编码、描述和状态
4. 点击「编辑」按钮，可以修改科室信息
5. 点击「删除」按钮，可以删除科室
6. 使用搜索框可以根据科室名称、编码和状态筛选科室

#### 4.1.2 证书管理
1. 登录系统后，点击左侧菜单的「管理员功能」→「证书管理」
2. 在证书管理页面，可以查看所有证书列表
3. 点击「新增证书」按钮，填写证书信息，包括证书名称、描述、及格分数、有效期、所属科目和状态
4. 点击「编辑」按钮，可以修改证书信息
5. 点击「删除」按钮，可以删除证书
6. 使用搜索框可以根据证书名称和状态筛选证书

#### 4.1.3 学习资源管理
1. 登录系统后，点击左侧菜单的「管理员功能」→「学习资源管理」
2. 在学习资源管理页面，可以查看所有学习资源列表
3. 点击「新增资源」按钮，填写资源信息，包括资源标题、描述、类型、链接、文件类型、文件大小、所属科目和状态
4. 点击「编辑」按钮，可以修改资源信息
5. 点击「预览」按钮，可以预览资源
6. 点击「删除」按钮，可以删除资源
7. 使用搜索框可以根据资源标题、类型和状态筛选资源

### 4.2 学员操作指南

#### 4.2.1 在线考试
1. 登录系统后，点击左侧菜单的「学员功能」→「考试中心」
2. 在考试中心页面，可以查看所有可用的考试
3. 点击「参加考试」按钮，进入考试页面
4. 按照考试要求完成答题
5. 点击「提交考试」按钮，提交考试答案

#### 4.2.2 练习功能
1. 登录系统后，点击左侧菜单的「学员功能」→「练习中心」
2. 在练习中心页面，可以选择不同的科目进行练习
3. 完成练习后，可以查看练习结果和解析

#### 4.2.3 错题集
1. 登录系统后，点击左侧菜单的「学员功能」→「错题集」
2. 在错题集页面，可以查看所有做错的题目
3. 点击题目可以查看题目详情和解析
4. 可以重新练习错题

## 5. 系统部署

### 5.1 Docker部署（推荐）

```bash
# 构建并启动所有服务
docker compose up -d

# 查看服务状态
docker compose ps

# 停止服务
docker compose down
```

### 5.2 手动部署

#### 5.2.1 后端部署

```bash
# 进入后端目录
cd server

# 构建jar包
mvn clean package -DskipTests

# 运行jar包
java -jar target/backend-0.0.1-SNAPSHOT.jar
```

#### 5.2.2 前端部署

```bash
# 进入前端目录
cd client

# 安装依赖
npm install

# 构建静态文件
npm run build

# 部署到Nginx
# 将dist目录下的文件复制到Nginx的web目录
```

### 5.3 数据库配置

1. 创建数据库：`smart_exam`
2. 创建用户：`smart_exam_user`
3. 赋予权限：`GRANT ALL PRIVILEGES ON smart_exam.* TO 'smart_exam_user'@'localhost' IDENTIFIED BY 'your-password' WITH GRANT OPTION;`
4. 刷新权限：`FLUSH PRIVILEGES;`
5. 修改 `server/src/main/resources/application.properties` 中的数据库连接配置

## 6. 系统配置

### 6.1 后端配置

后端配置文件位于 `server/src/main/resources/application.properties`，主要配置项包括：

- 应用名称和端口
- 数据库连接信息
- JPA配置
- Hikari连接池配置
- 日志配置

### 6.2 前端配置

前端配置文件位于 `client/vite.config.ts`，主要配置项包括：

- 开发服务器配置
- 代理配置
- 构建优化配置
- 代码分割配置

## 7. 系统维护

### 7.1 数据备份

定期备份数据库，确保数据安全。可以使用以下命令备份数据库：

```bash
docker exec -it smart-exam-mysql mysqldump -u smart_exam_user -p smart_exam > smart_exam_backup.sql
```

### 7.2 日志查看

查看后端日志：

```bash
docker logs -f smart-exam-app
```

查看MySQL日志：

```bash
docker logs -f smart-exam-mysql
```

### 7.3 常见问题及解决方案

#### 7.3.1 端口被占用

如果出现端口被占用的错误，可以修改 `docker-compose.yml` 文件中的端口映射：

```yaml
# 将宿主机的8080端口映射到容器的8080端口
# 可以修改宿主机端口，例如：8081:8080
ports:
  - "8080:8080"
```

#### 7.3.2 数据库连接失败

检查数据库配置是否正确，包括：
- 数据库地址
- 数据库名称
- 用户名和密码

#### 7.3.3 服务启动缓慢

首次启动时，需要拉取镜像和构建应用，可能会比较缓慢，请耐心等待。

#### 7.3.4 前端页面无法访问

检查：
- 后端服务是否正常运行
- 前端静态文件是否正确部署
- 浏览器控制台是否有错误信息

## 8. 开发指南

### 8.1 后端开发

进入后端目录，运行：

```bash
mvn spring-boot:run
```

### 8.2 前端开发

进入前端目录，运行：

```bash
npm run dev
```

前端开发服务器将在 `http://localhost:3000` 上运行。
